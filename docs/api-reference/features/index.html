Features

SourceScribe Documentation

SourceScribe is a powerful documentation generation tool that automatically creates user-centric, process-oriented documentation from source code. It supports a variety of AI-powered language models to generate feature descriptions, code examples, and visual diagrams.

Configuration Management​

Feature Overview​

SourceScribe's configuration management system allows users to customize the behavior of the documentation generation process. This includes setting up API credentials, defining file exclusions, and configuring the output format.

How It Works​

The configuration is loaded from a combination of environment variables, a YAML configuration file, and default settings. The sourcescribe.config module handles the loading and validation of these configuration options.

Use mouse wheel to zoom • Click and drag to pan

Usage Examples​

To set up a custom configuration, create a sourcescribe.yml file in your project directory with the following structure:

api:
  provider: openai
  api_key: your_openai_api_key

exclusions:
  - "**/__pycache__/**"
  - "**/*.pyc"

output:
  format: markdown
  directory: docs

Alternatively, you can set the configuration via environment variables:

export SOURCESCRIBE_API_PROVIDER=openai
export SOURCESCRIBE_API_KEY=your_openai_api_key
export SOURCESCRIBE_EXCLUSIONS='["**/__pycache__/**", "**/*.pyc"]'
export SOURCESCRIBE_OUTPUT_FORMAT=markdown
export SOURCESCRIBE_OUTPUT_DIRECTORY=docs

Configuration Options​

OptionDescriptionDefault
api.providerThe AI language model provider to use (e.g., openai, anthropic, ollama)openai
api.api_keyThe API key for the selected providerNone
exclusionsA list of glob patterns to exclude from the documentation generation[]
output.formatThe output format for the generated documentation (e.g., markdown, html)markdown
output.directoryThe directory to save the generated documentationdocs

API Integration​

Feature Overview​

SourceScribe integrates with various AI language models to generate feature descriptions, code examples, and visual diagrams. Currently, it supports OpenAI, Anthropic, and Ollama as API providers.

How It Works​

The sourcescribe.api module handles the communication with the selected API provider. It abstracts the provider-specific implementation details and provides a consistent interface for the rest of the application.

Use mouse wheel to zoom • Click and drag to pan

Usage Examples​

To use the API integration, you'll need to configure the provider and API key in your SourceScribe configuration:

from sourcescribe.api.factory import APIFactory

# Create an API provider instance
provider = APIFactory.get_provider()

# Generate feature description
feature_description = provider.generate_feature_description(source_code)

# Generate code example
code_example = provider.generate_code_example(source_code)

# Generate diagram
diagram = provider.generate_diagram(source_code)

Supported Providers​

  • OpenAI: Provides access to the OpenAI language models, such as GPT-3.
  • Anthropic: Provides access to the Anthropic language models, such as Claude.
  • Ollama: Provides access to the Ollama language models.

File Processing​

Feature Overview​

SourceScribe analyzes the source code files in your project and generates documentation based on the code structure and content. It supports various file types and can be configured to exclude certain files or directories.

How It Works​

The sourcescribe.engine module is responsible for the file processing and documentation generation. It uses the sourcescribe.utils.parser module to extract relevant information from the source code, and the sourcescribe.api module to generate the content using the selected AI language model.

Use mouse wheel to zoom • Click and drag to pan

Usage Examples​

To generate documentation for your project, run the SourceScribe CLI command:

sourcescribe generate

This will analyze the source code files in your project, generate feature descriptions, code examples, and diagrams, and save the documentation in the configured output directory.

Monitoring & Logging​

Feature Overview​

SourceScribe provides logging and monitoring capabilities to help users understand the documentation generation process and troubleshoot any issues that may arise.

How It Works​

The sourcescribe.utils.logger module is responsible for logging messages at different levels (debug, info, warning, error, critical). The logging configuration can be customized by setting the appropriate environment variables.

Use mouse wheel to zoom • Click and drag to pan

Usage Examples​

To configure the logging level, set the SOURCESCRIBE_LOG_LEVEL environment variable:

export SOURCESCRIBE_LOG_LEVEL=debug

This will enable debug-level logging, which can be useful for troubleshooting issues.

You can also configure the logging output format and destination by setting the following environment variables:

export SOURCESCRIBE_LOG_FORMAT="%(asctime)s %(levelname)s: %(message)s"
export SOURCESCRIBE_LOG_FILE="sourcescribe.log"

This will write the logs to a file named sourcescribe.log in the current directory.

Configuration Options​

OptionDescriptionDefault
SOURCESCRIBE_LOG_LEVELThe logging level (debug, info, warning, error, critical)info
SOURCESCRIBE_LOG_FORMATThe format of the log messages%(asctime)s %(levelname)s: %(message)s
SOURCESCRIBE_LOG_FILEThe path to the log file (leave empty for console output)None

Common Use Cases​

Automated Documentation Generation​

SourceScribe can be integrated into your development workflow to automatically generate up-to-date documentation for your project. This can be done by running the sourcescribe generate command as part of your CI/CD pipeline or as a pre-commit hook.

Exploratory Code Analysis​

When working with an unfamiliar codebase, SourceScribe can help you quickly understand the project's features and how they are implemented. The generated documentation, including feature descriptions, code examples, and diagrams, can provide valuable insights and serve as a starting point for further exploration.

Onboarding New Developers​

SourceScribe's user-centric documentation can be a valuable resource for onboarding new developers to your project. The clear explanations and visual diagrams can help them quickly grasp the project's structure and functionality, reducing the ramp-up time.

Documenting Legacy Systems​

Maintaining documentation for legacy systems can be a challenge, especially when the original developers are no longer available. SourceScribe can analyze the existing codebase and generate comprehensive documentation, making it easier to understand and maintain the system.

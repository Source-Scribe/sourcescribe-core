Features

SourceScribe Documentation

SourceScribe is a powerful documentation generation tool that analyzes source code and automatically generates user-friendly documentation. It supports a variety of programming languages and integrates with popular AI-powered writing assistants to enhance the documentation process.

Features​

SourceScribe provides the following key features:

  1. Code Analysis & Documentation Generation
  2. AI-Powered Writing Assistance
  3. Customizable Configuration
  4. Monitoring & Logging
  5. File Watcher & Automatic Updates

Code Analysis & Documentation Generation​

SourceScribe analyzes your source code and generates comprehensive documentation, including:

  • Feature Extraction: Identifies and documents the key features and capabilities of your codebase.
  • Workflow Diagrams: Generates sequence diagrams and flowcharts to illustrate how the system works.
  • Data Models: Creates class diagrams to visualize the data structures and relationships.
  • Code Examples: Provides practical code samples to demonstrate feature usage.

How It Works​

The code analysis and documentation generation process in SourceScribe follows these steps:

Use mouse wheel to zoom • Click and drag to pan
  1. Code Parser: SourceScribe's parser module analyzes the source code and extracts relevant information, such as function definitions, class structures, and data types.
  2. Feature Extractor: The feature extraction module identifies the key capabilities and features of the codebase based on the parsed information.
  3. Diagram Generator: SourceScribe generates various types of diagrams, including sequence diagrams, flowcharts, and class diagrams, to visually represent the system's workflows and data models.
  4. Document Generator: The document generation module combines the extracted features, diagrams, and other relevant information to create the final documentation.

Usage Examples​

To generate documentation for your project, you can use the SourceScribe command-line interface (CLI) or integrate it into your build process. Here's an example of how to use the CLI:

from sourcescribe.cli import generate_documentation

generate_documentation(
    input_dir="path/to/your/codebase",
    output_dir="path/to/output/directory",
    config_file="path/to/config.yaml"
)

This will analyze the source code in the input_dir directory, generate the documentation, and save it in the output_dir directory. The config_file parameter allows you to customize the documentation generation process.

Configuration​

SourceScribe provides several configuration options to customize the documentation generation process. Some of the key settings include:

SettingDescription
input_dirThe directory containing the source code to be analyzed.
output_dirThe directory where the generated documentation will be saved.
excluded_filesA list of files or directories to be excluded from the analysis.
ai_providerThe AI writing assistant to be used for content generation (e.g., OpenAI, Anthropic).
api_keyThe API key for the selected AI writing assistant.

You can specify these settings in a YAML configuration file and pass it to the generate_documentation function.

Common Use Cases​

SourceScribe can be used in a variety of scenarios, including:

  1. Onboarding New Developers: Automatically generate comprehensive documentation to help new team members quickly understand the codebase and its features.
  2. Open-Source Project Documentation: Streamline the documentation process for open-source projects, making it easier for contributors and users to engage with the project.
  3. Internal Documentation: Use SourceScribe to maintain up-to-date, user-friendly documentation for internal software systems and tools.
  4. Client Deliverables: Provide clients with detailed, visually-appealing documentation as part of your software delivery process.

AI-Powered Writing Assistance​

SourceScribe integrates with popular AI writing assistants, such as OpenAI and Anthropic, to enhance the documentation generation process. These AI models can be used to:

  • Content Generation: Automatically generate descriptive text for features, workflows, and usage examples.
  • Tone and Style Optimization: Ensure the documentation is written in a clear, user-friendly tone.
  • Grammar and Spelling Correction: Improve the overall quality and readability of the documentation.

How It Works​

SourceScribe's AI integration follows these steps:

Use mouse wheel to zoom • Click and drag to pan
  1. The user initiates the documentation generation process using SourceScribe.
  2. SourceScribe sends a request to the selected AI provider (e.g., OpenAI, Anthropic) to generate content for specific sections of the documentation.
  3. The AI provider processes the request and returns the generated content to SourceScribe.
  4. SourceScribe incorporates the AI-generated content into the final documentation.
  5. SourceScribe presents the completed documentation to the user.

Configuration​

To enable AI-powered writing assistance, you'll need to configure the following settings in your SourceScribe configuration file:

SettingDescription
ai_providerThe AI writing assistant to be used (e.g., "openai", "anthropic").
api_keyThe API key for the selected AI writing assistant.
ai_content_generation_enabledWhether to use the AI provider for content generation (true/false).
ai_tone_optimization_enabledWhether to use the AI provider for tone and style optimization (true/false).
ai_grammar_correction_enabledWhether to use the AI provider for grammar and spelling correction (true/false).

Make sure to obtain the necessary API keys from the AI providers and securely store them in your configuration file.

Common Use Cases​

AI-powered writing assistance in SourceScribe can be beneficial in the following scenarios:

  1. Rapid Documentation Generation: Leverage AI models to quickly generate initial drafts of the documentation, reducing the time and effort required.
  2. Consistent Tone and Style: Ensure the documentation maintains a clear, user-friendly tone and style across all sections.
  3. Improved Readability: Use AI-powered grammar and spelling correction to enhance the overall quality and readability of the documentation.
  4. Multilingual Documentation: Explore the use of AI translation services to generate documentation in multiple languages.

Customizable Configuration​

SourceScribe provides a flexible configuration system that allows you to customize the documentation generation process to suit your specific needs.

How It Works​

SourceScribe's configuration system is based on a YAML file that specifies various settings and options. The configuration file is loaded and parsed by the sourcescribe.config.loader module, which then applies the settings throughout the documentation generation process.

Use mouse wheel to zoom • Click and drag to pan
  1. The user initiates the documentation generation process and provides the path to the configuration file.
  2. The ConfigLoader module reads and parses the YAML configuration file.
  3. The ConfigLoader passes the parsed configuration settings to the DocumentationGenerator.
  4. The DocumentationGenerator uses the provided configuration to generate the final documentation.
  5. The completed documentation is presented to the user.

Configuration Options​

The SourceScribe configuration file supports the following key settings:

SettingDescription
input_dirThe directory containing the source code to be analyzed.
output_dirThe directory where the generated documentation will be saved.
excluded_filesA list of files or directories to be excluded from the analysis.
ai_providerThe AI writing assistant to be used for content generation (e.g., "openai", "anthropic").
api_keyThe API key for the selected AI writing assistant.
ai_content_generation_enabledWhether to use the AI provider for content generation (true/false).
ai_tone_optimization_enabledWhether to use the AI provider for tone and style optimization (true/false).
ai_grammar_correction_enabledWhether to use the AI provider for grammar and spelling correction (true/false).

You can customize these settings to fit your specific documentation requirements and preferences.

Common Use Cases​

The customizable configuration in SourceScribe can be useful in the following scenarios:

  1. Tailored Documentation: Adjust the configuration to generate documentation that aligns with your organization's branding, tone, and style guidelines.
  2. Selective Documentation: Use the excluded_files setting to generate documentation for a specific subset of your codebase, rather than the entire project.
  3. Automated Integration: Integrate the configuration settings into your build or deployment process to generate up-to-date documentation as part of your software development lifecycle.
  4. Multilingual Support: Explore the use of AI translation services to generate documentation in multiple languages by configuring the appropriate settings.

Monitoring & Logging​

SourceScribe provides robust monitoring and logging capabilities to help you track the documentation generation process and troubleshoot any issues that may arise.

How It Works​

SourceScribe's monitoring and logging functionality is powered by the sourcescribe.utils.logger module, which uses the standard Python logging library. The logger is configured to output messages to both the console and a log file, with customizable log levels and formatting.

Use mouse wheel to zoom • Click and drag to pan
  1. The user initiates the documentation generation process using SourceScribe.
  2. As the DocumentationGenerator module executes, it sends log messages to the Logger module.
  3. The Logger module processes the log messages and outputs them to both the console and a log file.
  4. The completed documentation is presented to the user, along with the log output for monitoring and troubleshooting purposes.

Configuration​

The logging configuration in SourceScribe can be customized through the following settings in the YAML configuration file:

SettingDescription
log_levelThe minimum log level to be recorded (e.g., "DEBUG", "INFO", "WARNING", "ERROR").
log_fileThe path to the log file where the log messages will be saved.
log_formatThe formatting pattern for the log messages.

You can adjust these settings to control the level of detail and the output location of the log messages.

Common Use Cases​

The monitoring and logging capabilities in SourceScribe can be useful in the following scenarios:

  1. Troubleshooting: Use the log output to identify and address any issues that may occur during the documentation generation process.
  2. Process Monitoring: Monitor the progress and status of the documentation generation, especially for large or complex codebases.
  3. Auditing: Maintain a record of the documentation generation activities for compliance or reporting purposes.
  4. Continuous Integration: Integrate the logging output into your CI/CD pipeline to track the documentation generation as part of your software build and deployment process.

File Watcher & Automatic Updates​

SourceScribe includes a file watcher functionality that automatically detects changes in the source code and triggers the documentation generation process, ensuring that the documentation remains up-to-date.

How It Works​

The file watcher in SourceScribe is implemented using the sourcescribe.watch.watcher module, which continuously monitors the specified input directory for changes. When a change is detected, the watcher triggers the sourcescribe.cli.generate_documentation function to regenerate the documentation.

Use mouse wheel to zoom • Click and drag to pan
  1. The user starts the file watcher using SourceScribe.
  2. The Watcher module continuously monitors the input directory for changes.
  3. When a change is detected, the Watcher triggers the DocumentationGenerator to regenerate the documentation.
  4. The DocumentationGenerator produces the updated documentation and returns it to the Watcher.
  5. The Watcher notifies the user that the documentation has been updated.

Configuration​

The file watcher in SourceScribe can be configured through the following settings in the YAML configuration file:

SettingDescription
watch_enabledWhether the file watcher functionality is enabled (true/false).
watch_directoryThe directory to be monitored for changes.
watch_intervalThe interval (in seconds) at which the directory is checked for changes.
watch_extensionsA list of file extensions to be monitored for changes.

You can adjust these settings to control the behavior of the file watcher and the documentation generation trigger.

Common Use Cases​

The file watcher and automatic update functionality in SourceScribe can be useful in the following scenarios:

  1. Continuous Documentation Updates: Ensure that the documentation is always up-to-date with the latest changes in the source code, without requiring manual intervention.
  2. Developer Productivity: Eliminate the need for developers to manually generate documentation after making changes to the codebase.
  3. Collaboration and Sharing: Provide team members with immediate access to the latest version of the documentation, fostering better collaboration and knowledge sharing.
  4. Deployment Automation: Integrate the file watcher into your CI/CD pipeline to automatically generate and deploy updated documentation as part of your software release process.

Conclusion​

SourceScribe is a powerful documentation generation tool that simplifies the process of creating user-centric, process-oriented documentation for your software projects. By leveraging advanced features like AI-powered writing assistance, customizable configuration, and automatic file monitoring, SourceScribe helps you deliver high-quality documentation that enhances developer productivity and collaboration.

Whether you're working on an open-source project, maintaining an internal software system, or delivering client-facing documentation, SourceScribe can streamline your documentation efforts and ensure that your codebase is well-documented and easily accessible to all stakeholders.

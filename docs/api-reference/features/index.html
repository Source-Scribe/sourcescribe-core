Features

SourceScribe Documentation

SourceScribe is a powerful documentation generation tool that automatically creates user-friendly technical documentation from source code. It supports a variety of programming languages and integrates with popular AI-powered writing assistants to generate high-quality, process-oriented documentation.

Features​

SourceScribe provides the following key features:

  1. Source Code Analysis: Analyze source code files to extract relevant information for documentation generation.
  2. AI-Powered Content Generation: Leverage AI writing assistants like OpenAI and Anthropic to generate natural language descriptions and explanations.
  3. Diagram Generation: Automatically create visual diagrams, such as sequence diagrams, flowcharts, and state diagrams, to illustrate key concepts and workflows.
  4. Configuration Management: Flexible configuration system to customize the documentation generation process.
  5. Monitoring & Logging: Track the documentation generation process and handle errors or issues.
  6. Extensible Architecture: Easily integrate new AI providers or extend the functionality of SourceScribe.

Let's dive into each of these features in more detail.

Source Code Analysis​

SourceScribe analyzes the source code files in your project to extract relevant information for documentation generation. This includes:

  • Identifying code entities (functions, classes, modules, etc.)
  • Parsing code comments and docstrings
  • Detecting code dependencies and relationships

How It Works:

  1. The SourceScribeAnalyzer class scans the source code files in the project.
  2. It parses the code to extract relevant metadata, such as function signatures, class definitions, and code comments.
  3. The extracted information is stored in an internal data model, which is then used by the documentation generation engine.

Usage Example:

from sourcescribe.engine.analyzer import SourceScribeAnalyzer

analyzer = SourceScribeAnalyzer()
analyzer.analyze("path/to/your/project")

Configuration:

  • source_dir: The directory containing the source code files to analyze.
  • exclude_patterns: A list of file patterns to exclude from the analysis.

AI-Powered Content Generation​

SourceScribe integrates with AI writing assistants, such as OpenAI and Anthropic, to generate natural language descriptions and explanations for the documented code entities.

How It Works:

  1. The FeatureGenerator class collects the relevant information about a code entity (e.g., function, class, module).
  2. It then sends a prompt to the configured AI provider, requesting a natural language description of the entity.
  3. The AI provider processes the prompt and returns the generated content, which is then incorporated into the documentation.

Usage Example:

from sourcescribe.engine.feature_generator import FeatureGenerator
from sourcescribe.api.factory import AIProviderFactory

provider = AIProviderFactory.create("openai")
generator = FeatureGenerator(provider)
description = generator.generate_description(my_function)

Configuration:

  • ai_provider: The AI writing assistant to use (e.g., "openai", "anthropic").
  • api_key: The API key for the configured AI provider.

Diagram Generation​

SourceScribe automatically generates various types of diagrams, such as sequence diagrams, flowcharts, and state diagrams, to visually illustrate key concepts and workflows.

How It Works:

  1. The DiagramGenerator class analyzes the code and data model to identify the appropriate diagram type for a given feature or workflow.
  2. It then generates the diagram using the Mermaid.js library and includes it in the documentation.

Usage Example:

from sourcescribe.engine.diagram import DiagramGenerator

diagram_generator = DiagramGenerator()
sequence_diagram = diagram_generator.generate_sequence_diagram(my_workflow)

Configuration:

  • diagram_types: A list of diagram types to generate (e.g., "sequence", "flowchart", "state").
  • diagram_quality: The quality level for the generated diagrams (e.g., "high", "medium", "low").

Configuration Management​

SourceScribe provides a flexible configuration system to customize the documentation generation process.

How It Works:

  1. The ConfigLoader class loads the configuration settings from various sources (e.g., YAML file, environment variables).
  2. The ConfigModel class defines the structure and validation rules for the configuration settings.
  3. The configuration is used throughout the documentation generation process to control various aspects, such as AI provider selection, diagram generation, and file exclusions.

Usage Example:

from sourcescribe.config.loader import ConfigLoader
from sourcescribe.config.models import SourceScribeConfig

config = ConfigLoader.load("path/to/config.yml")
print(config.ai_provider)
print(config.diagram_types)

Configuration:

  • config_file: The path to the YAML configuration file.
  • env_prefix: The prefix for environment variables to be used as configuration.

Monitoring & Logging​

SourceScribe provides logging and monitoring capabilities to track the documentation generation process and handle errors or issues.

How It Works:

  1. The Logger class is used throughout the SourceScribe codebase to log various events and messages.
  2. The WatchHandler class monitors the documentation generation process and reports any errors or issues that occur.
  3. The logged information can be used to troubleshoot problems and improve the documentation generation process.

Usage Example:

from sourcescribe.utils.logger import Logger

logger = Logger()
logger.info("Starting documentation generation...")
try:
    # Generate documentation
except Exception as e:
    logger.error(f"Error occurred: {e}")

Configuration:

  • log_level: The minimum log level to record (e.g., "DEBUG", "INFO", "WARNING", "ERROR").
  • log_file: The path to the log file.

Extensible Architecture​

SourceScribe is designed with an extensible architecture, allowing you to easily integrate new AI providers or extend the functionality of the tool.

How It Works:

  1. The AIProviderFactory class is responsible for creating instances of AI providers.
  2. To integrate a new AI provider, you can create a new class that implements the BaseAIProvider interface and register it with the factory.
  3. Similarly, you can extend the functionality of SourceScribe by creating new classes that implement the desired features and integrating them into the existing codebase.

Usage Example:

from sourcescribe.api.factory import AIProviderFactory
from sourcescribe.api.my_custom_provider import MyCustomProvider

AIProviderFactory.register("my_custom_provider", MyCustomProvider)
provider = AIProviderFactory.create("my_custom_provider")

Configuration:

  • No additional configuration is required for the extensible architecture.

Common Use Cases​

Here are some common use cases for SourceScribe:

  1. Automated Documentation Generation: Generate comprehensive, user-centric documentation for your software projects without the need for manual writing.
  2. Onboarding and Training: Provide new team members with detailed, visually-rich documentation to help them quickly understand and use your codebase.
  3. Code Comprehension: Leverage the generated diagrams and explanations to better understand complex codebases, especially when working on legacy projects.
  4. Collaboration and Knowledge Sharing: Share the generated documentation with stakeholders, clients, or the wider community to facilitate collaboration and knowledge sharing.
  5. Maintenance and Refactoring: Use the documentation to guide maintenance and refactoring efforts, ensuring that changes are made with a clear understanding of the system's architecture and workflows.

Conclusion​

SourceScribe is a powerful documentation generation tool that simplifies the process of creating user-friendly, process-oriented technical documentation. By leveraging AI-powered content generation, automated diagram creation, and a flexible configuration system, SourceScribe helps you deliver high-quality documentation that enhances code comprehension, collaboration, and maintenance.

Features

Source Scribe Documentation

Source Scribe is a powerful documentation generation tool that automatically creates user-centric, process-oriented documentation from your codebase. It leverages advanced language models and code analysis techniques to generate high-quality documentation with extensive visual diagrams.

Features​

Source Scribe provides the following key features:

  1. Configuration Management
  2. API Integration
  3. File Processing
  4. Monitoring & Logging

Configuration Management​

Feature Overview​

The Configuration Management feature in Source Scribe allows you to define and manage the configuration settings for your documentation generation process. This includes specifying the input source code directories, output paths, and various options to customize the documentation output.

How It Works​

The configuration management in Source Scribe is powered by the config/models.py module, which defines the configuration data model using Pydantic. The config/loader.py module is responsible for loading the configuration from various sources, such as environment variables, a configuration file, or command-line arguments.

Usage Examples​

Here's an example of how to use the Configuration Management feature:

from sourcescribe.config.models import SourceScribeConfig

config = SourceScribeConfig(
    input_dir="path/to/your/codebase",
    output_dir="path/to/output",
    api_provider="openai",
    api_key="your_api_key",
    # Other configuration options
)

You can also load the configuration from a YAML file:

from sourcescribe.config.loader import load_config

config = load_config("path/to/config.yml")

Configuration​

The following configuration options are available:

OptionDescriptionDefault
input_dirThe directory containing the source code to be documented."."
output_dirThe directory where the generated documentation will be saved."docs"
api_providerThe API provider to use for generating documentation. Supported values: "openai", "anthropic", "ollama"."openai"
api_keyThe API key for the selected provider.""
exclusionsA list of file patterns to exclude from the documentation generation.[]
max_file_sizeThe maximum file size (in bytes) to process. Larger files will be skipped.1000000
max_line_countThe maximum number of lines in a file to process. Larger files will be skipped.10000
max_request_tokensThe maximum number of tokens to include in a single API request.4096
max_response_tokensThe maximum number of tokens to accept in a single API response.8192

Common Use Cases​

  1. Generating Documentation for a Specific Codebase: Configure the input_dir and output_dir options to generate documentation for your project's source code.
  2. Excluding Specific Files or Directories: Use the exclusions option to prevent certain files or directories from being processed.
  3. Integrating with Different AI Providers: Change the api_provider and api_key options to use a different AI provider for the documentation generation.
  4. Adjusting Performance Limits: Modify the max_file_size, max_line_count, max_request_tokens, and max_response_tokens options to optimize the performance of the documentation generation process.

API Integration​

Feature Overview​

The API Integration feature in Source Scribe allows you to generate documentation using various AI-powered language models, such as OpenAI, Anthropic, and Ollama. This feature abstracts the underlying API calls and provides a consistent interface for interacting with different providers.

How It Works​

The API Integration feature is implemented in the api/ module. The api/base.py module defines the base BaseProvider class, which provides the common interface for all API providers. The specific provider implementations, such as api/openai_provider.py, api/anthropic_provider.py, and api/ollama_provider.py, extend this base class and implement the provider-specific logic.

Use mouse wheel to zoom • Click and drag to pan

Usage Examples​

Here's an example of how to use the API Integration feature:

from sourcescribe.api.factory import get_provider
from sourcescribe.config.models import SourceScribeConfig

config = SourceScribeConfig(
    api_provider="openai",
    api_key="your_openai_api_key"
)

provider = get_provider(config)
documentation = provider.generate_documentation(code, config)

You can easily switch between different API providers by changing the api_provider option in the configuration.

Configuration​

The API Integration feature is configured through the following options in the SourceScribeConfig model:

OptionDescriptionDefault
api_providerThe API provider to use for generating documentation. Supported values: "openai", "anthropic", "ollama"."openai"
api_keyThe API key for the selected provider.""

Common Use Cases​

  1. Generating Documentation with OpenAI: Use the OpenAI provider by setting api_provider="openai" and providing your OpenAI API key.
  2. Generating Documentation with Anthropic: Use the Anthropic provider by setting api_provider="anthropic" and providing your Anthropic API key.
  3. Generating Documentation with Ollama: Use the Ollama provider by setting api_provider="ollama" and providing your Ollama API key.
  4. Switching Between Providers: Easily change the api_provider option to use a different language model for generating the documentation.

File Processing​

Feature Overview​

The File Processing feature in Source Scribe is responsible for analyzing the source code files, extracting relevant information, and preparing the data for the documentation generation process. This includes tasks such as parsing the code, identifying code structures, and handling various file types.

How It Works​

The File Processing feature is implemented in the engine/ module. The engine/analyzer.py module is responsible for analyzing the source code files and extracting the necessary information, such as function definitions, class structures, and variable declarations. The engine/feature_generator.py module then processes the extracted information and generates the documentation features.

Usage Examples​

The File Processing feature is primarily used internally by the Source Scribe application. However, you can access the underlying functionality if needed:

from sourcescribe.engine.analyzer import analyze_files
from sourcescribe.engine.feature_generator import generate_features

# Analyze the source code files
analyzed_data = analyze_files(input_dir, config)

# Generate the documentation features
features = generate_features(analyzed_data, config)

Configuration​

The File Processing feature is configured through the following options in the SourceScribeConfig model:

OptionDescriptionDefault
input_dirThe directory containing the source code to be documented."."
exclusionsA list of file patterns to exclude from the documentation generation.[]
max_file_sizeThe maximum file size (in bytes) to process. Larger files will be skipped.1000000
max_line_countThe maximum number of lines in a file to process. Larger files will be skipped.10000

Common Use Cases​

  1. Analyzing a Codebase: Use the analyze_files function to extract information from the source code files in your project.
  2. Generating Documentation Features: Utilize the generate_features function to create the necessary data structures for the documentation generation process.
  3. Excluding Specific Files or Directories: Configure the exclusions option to prevent certain files or directories from being processed.
  4. Handling Large Files: Adjust the max_file_size and max_line_count options to optimize the performance of the file processing step.

Monitoring & Logging​

Feature Overview​

The Monitoring & Logging feature in Source Scribe provides comprehensive logging and error handling capabilities to help you understand the documentation generation process and troubleshoot any issues that may arise.

How It Works​

The Monitoring & Logging feature is implemented in the utils/logger.py module, which sets up a centralized logging system for the entire application. The logging system uses the standard Python logging module and provides various log levels (debug, info, warning, error, critical) to help you monitor the application's behavior.

Use mouse wheel to zoom • Click and drag to pan

Usage Examples​

You can access the logging functionality throughout the Source Scribe codebase. Here's an example of how to use it:

from sourcescribe.utils.logger import get_logger

logger = get_logger(__name__)

try:
    # Some code that might raise an exception
    pass
except Exception as e:
    logger.error("An error occurred: %s", e, exc_info=True)

Configuration​

The Monitoring & Logging feature does not have any specific configuration options. However, you can configure the logging behavior by setting environment variables or using a configuration file. For example, you can set the LOGLEVEL environment variable to control the log level:

export LOGLEVEL=DEBUG

Common Use Cases​

  1. Debugging Issues: Use the logger.debug() and logger.error() functions to log relevant information during the documentation generation process, which can help you identify and fix any issues that arise.
  2. Monitoring the Application: Monitor the log output to understand the overall behavior of the Source Scribe application and track its progress.
  3. Integrating with External Logging Systems: Configure the logging system to send log messages to external logging services, such as Syslog or Splunk, for centralized monitoring and analysis.

Conclusion​

Source Scribe is a powerful documentation generation tool that provides a comprehensive set of features to help you create user-centric, process-oriented documentation for your codebase. By leveraging advanced language models and code analysis techniques, Source Scribe can generate high-quality documentation with extensive visual diagrams, making it easier for developers to understand and use your software.

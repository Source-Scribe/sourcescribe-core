Features

SourceScribe Documentation

SourceScribe is a powerful documentation generation tool that automatically creates user-friendly technical documentation from source code. It supports a variety of programming languages and integrates with popular AI-powered writing assistants to generate high-quality, process-oriented documentation.

Features​

SourceScribe provides the following key features:

  1. Code Analysis: Analyze source code to extract relevant information for documentation generation.
  2. AI-Powered Writing: Leverage AI-based writing assistants to generate natural language descriptions and explanations.
  3. Diagram Generation: Automatically create visual diagrams, such as sequence diagrams, flowcharts, and class diagrams, to illustrate key concepts and workflows.
  4. Configuration Management: Flexible configuration options to customize the documentation generation process.
  5. Monitoring & Logging: Track the documentation generation process and handle errors or issues.
  6. Extensibility: Easily integrate with new AI providers or extend the core functionality.

Let's dive into each of these features in more detail.

Code Analysis​

SourceScribe analyzes the source code to extract relevant information for documentation generation. This includes:

  • Parsing the code structure (functions, classes, modules, etc.)
  • Identifying code comments and docstrings
  • Detecting dependencies and relationships between code elements

How It Works

  1. The SourceScribe engine reads the source code files.
  2. The Parser module extracts relevant information from the code, such as function signatures, class definitions, and code comments.
  3. The Analyzer module processes the parsed data to identify key features, relationships, and dependencies.
  4. The extracted information is then used by the FeatureGenerator and Diagram modules to create the documentation content.

Usage Examples

from sourcescribe.engine.analyzer import Analyzer

analyzer = Analyzer()
analysis_result = analyzer.analyze_codebase("path/to/your/codebase")

# Access the extracted information
print(analysis_result.classes)
print(analysis_result.functions)
print(analysis_result.dependencies)

Configuration

The code analysis process can be customized using the following configuration options:

OptionDescriptionDefault
exclude_pathsList of paths to exclude from the analysis[]
exclude_filesList of file names to exclude from the analysis[]
exclude_patternsList of regular expression patterns to exclude files[]

Common Use Cases

  1. Automated Documentation Generation: SourceScribe can be used to generate comprehensive documentation for your project, reducing the manual effort required to maintain documentation.
  2. Code Comprehension: The code analysis features can help new developers quickly understand the structure and dependencies of a codebase.
  3. Refactoring and Maintenance: The extracted information can be used to identify areas of the codebase that need improvement or refactoring.

AI-Powered Writing​

SourceScribe integrates with AI-based writing assistants to generate natural language descriptions and explanations for the documented features and concepts.

How It Works

  1. The FeatureGenerator module extracts relevant information about the code, such as function signatures, class definitions, and code comments.
  2. The extracted information is then passed to the AIProvider module, which interfaces with the selected AI writing assistant (e.g., Anthropic, OpenAI, Ollama).
  3. The AI writing assistant generates natural language descriptions and explanations based on the provided information.
  4. The generated content is then incorporated into the final documentation.

Usage Examples

from sourcescribe.api.factory import AIProviderFactory

provider = AIProviderFactory.create("openai")
description = provider.generate_description("my_function", "This function does something cool.")

Configuration

SourceScribe supports multiple AI writing assistants, which can be configured using the following options:

OptionDescriptionDefault
ai_providerThe AI writing assistant to use (e.g., "openai", "anthropic", "ollama")"openai"
ai_api_keyThe API key for the selected AI writing assistantNone
ai_temperatureThe temperature parameter for the AI writing assistant0.7

Common Use Cases

  1. Automated Documentation Generation: SourceScribe can use AI-powered writing to generate high-quality, natural language descriptions for your codebase, reducing the manual effort required.
  2. Improving Documentation Quality: The AI-generated content can help improve the clarity and consistency of your documentation, making it more user-friendly.
  3. Multilingual Documentation: SourceScribe can potentially be extended to support generating documentation in multiple languages using AI translation capabilities.

Diagram Generation​

SourceScribe automatically generates various types of diagrams, such as sequence diagrams, flowcharts, and class diagrams, to illustrate key concepts and workflows.

How It Works

  1. The Analyzer module extracts relevant information from the source code, such as function calls, class relationships, and control flow.
  2. The Diagram module uses this information to generate the appropriate diagram type (e.g., sequence diagram, flowchart, class diagram).
  3. The generated diagrams are then included in the final documentation.

Usage Examples

from sourcescribe.engine.diagram import DiagramGenerator

diagram_generator = DiagramGenerator()
sequence_diagram = diagram_generator.generate_sequence_diagram("my_module.my_function")
flowchart = diagram_generator.generate_flowchart("my_module.my_function")
class_diagram = diagram_generator.generate_class_diagram("my_module")

Configuration

The diagram generation process can be customized using the following configuration options:

OptionDescriptionDefault
diagram_themeThe theme to use for the generated diagrams"default"
diagram_font_sizeThe font size for the diagram elements14
diagram_widthThe width of the generated diagrams800
diagram_heightThe height of the generated diagrams600

Common Use Cases

  1. Improved Comprehension: The automatically generated diagrams can help users quickly understand the structure, relationships, and workflows within the codebase.
  2. Documentation Quality: Incorporating diagrams into the documentation can make it more visually appealing and easier to navigate.
  3. Collaboration and Onboarding: The diagrams can be used to facilitate discussions and help new team members understand the system.

Configuration Management​

SourceScribe provides a flexible configuration system to customize the documentation generation process.

How It Works

  1. The ConfigLoader module reads the configuration settings from a YAML or JSON file.
  2. The ConfigModel class defines the structure and validation rules for the configuration options.
  3. The configuration settings are then used by the various SourceScribe modules to control the documentation generation process.

Usage Examples

from sourcescribe.config.loader import ConfigLoader

config = ConfigLoader.load_config("path/to/config.yml")

# Access the configuration settings
print(config.ai_provider)
print(config.diagram_theme)
print(config.exclude_paths)

Configuration Options

Common Use Cases

  1. Customized Documentation: The configuration options allow users to tailor the documentation generation process to their specific needs, such as adjusting the diagram styles or excluding certain files from the analysis.
  2. Consistent Documentation: By storing the configuration settings in a central location, the documentation can be generated consistently across different environments or projects.
  3. Extensibility: The configuration system makes it easier to add new features or integrate with additional AI writing assistants in the future.

Monitoring & Logging​

SourceScribe provides logging and monitoring capabilities to track the documentation generation process and handle any errors or issues that may arise.

How It Works

  1. The Logger module is used throughout the SourceScribe codebase to log relevant information, warnings, and errors.
  2. The Watcher module monitors the documentation generation process, detecting any issues or errors that may occur.
  3. The Handler module processes the detected issues, providing appropriate error handling and reporting.

Usage Examples

from sourcescribe.utils.logger import Logger

Logger.info("Starting documentation generation process")

try:
    # Generate documentation
    pass
except Exception as e:
    Logger.error(f"Error generating documentation: {e}")

Configuration

The logging and monitoring features can be customized using the following configuration options:

OptionDescriptionDefault
log_levelThe minimum log level to record (e.g., "DEBUG", "INFO", "WARNING", "ERROR")"INFO"
log_fileThe path to the log file"sourcescribe.log"
watch_intervalThe interval (in seconds) for the watcher to check for issues60
watch_timeoutThe timeout (in seconds) for the watcher to consider an issue as resolved300

Common Use Cases

  1. Troubleshooting: The logging and monitoring features can help users identify and resolve issues that may occur during the documentation generation process.
  2. Auditing: The log files can be used to track the history of the documentation generation process and identify any changes or errors over time.
  3. Automated Monitoring: The watcher module can be integrated with external monitoring systems to provide real-time alerts and notifications about the documentation generation process.

Extensibility​

SourceScribe is designed to be easily extensible, allowing users to integrate new AI writing assistants or add custom functionality.

How It Works

  1. The AIProviderFactory class provides a pluggable interface for integrating new AI writing assistants.
  2. Users can create a new AIProvider implementation that conforms to the expected interface.
  3. The new AIProvider can then be registered with the AIProviderFactory, allowing it to be used throughout the SourceScribe application.

Usage Examples

from sourcescribe.api.base import AIProvider
from sourcescribe.api.factory import AIProviderFactory

class MyCustomAIProvider(AIProvider):
    def generate_description(self, entity, context):
        # Implement custom logic to generate description
        return "This is a custom description."

AIProviderFactory.register("my_custom_provider", MyCustomAIProvider)

# Use the custom AI provider
provider = AIProviderFactory.create("my_custom_provider")
description = provider.generate_description("my_entity", "some context")

Configuration

To use a custom AI provider, you can update the ai_provider configuration option to the name of your custom provider:

ai_provider: my_custom_provider

Common Use Cases

  1. Integrating New AI Assistants: SourceScribe can be extended to support additional AI writing assistants, such as custom or in-house solutions, by implementing a new AIProvider class.
  2. Customizing Diagram Generation: The Diagram module can be extended to support new diagram types or customize the generation of existing diagrams.
  3. Extending Core Functionality: The modular design of SourceScribe allows users to add new features or modify existing ones by creating custom modules or components.

By leveraging the extensibility of SourceScribe, users can tailor the tool to their specific needs and integrate it seamlessly into their existing development workflows.

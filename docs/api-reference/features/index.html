Features

SourceScribe Documentation

SourceScribe is a powerful documentation generation tool that automatically creates user-friendly technical documentation from source code. It supports a variety of programming languages and integrates with popular AI-powered writing assistants to generate high-quality, process-oriented documentation.

Features​

SourceScribe provides the following key features:

  1. Code Analysis: Analyze source code to extract relevant information for documentation generation.
  2. AI-Powered Writing: Leverage AI-based writing assistants like Anthropic and OpenAI to generate natural language content.
  3. Diagram Generation: Automatically create visual diagrams, such as sequence diagrams, flowcharts, and class diagrams, to illustrate key concepts.
  4. Configuration Management: Flexible configuration system to customize the documentation generation process.
  5. Monitoring & Logging: Track the documentation generation process and handle errors or issues.
  6. Extensibility: Easily integrate new AI providers or extend the core functionality.

Code Analysis​

SourceScribe analyzes the source code to extract relevant information for documentation generation. This includes:

  • Parsing the code structure and identifying key components
  • Extracting function/method signatures, parameters, and return types
  • Detecting data models and their relationships
  • Identifying control flow and decision points

How It Works​

Usage Examples​

from sourcescribe.engine.analyzer import CodeAnalyzer

analyzer = CodeAnalyzer()
metadata = analyzer.analyze_code('/path/to/source/code')
print(metadata)

Configuration​

The code analysis process can be configured using the following settings:

SettingDescriptionDefault Value
include_patternsList of file patterns to include in the analysis['*.py']
exclude_patternsList of file patterns to exclude from the analysis[]
max_file_sizeMaximum file size (in bytes) to analyze1048576 (1 MB)

Common Use Cases​

  1. Documenting a New Project: Use SourceScribe to automatically generate initial documentation for a new project, providing a solid foundation for further refinement.
  2. Maintaining Documentation: Integrate SourceScribe into your development workflow to keep the documentation up-to-date as the codebase evolves.
  3. Onboarding New Developers: Leverage the generated documentation to quickly onboard new team members and help them understand the project's architecture and functionality.

AI-Powered Writing​

SourceScribe integrates with AI-based writing assistants, such as Anthropic and OpenAI, to generate natural language content for the documentation. This includes:

  • High-level feature descriptions
  • Detailed explanations of how the system works
  • Practical usage examples
  • Contextual information and background

How It Works​

Use mouse wheel to zoom • Click and drag to pan

Usage Examples​

from sourcescribe.api.factory import AIProviderFactory

provider = AIProviderFactory.create('anthropic')
content = provider.generate_content('Explain the purpose of the SourceScribe tool')
print(content)

Configuration​

The AI-powered writing feature can be configured using the following settings:

SettingDescriptionDefault Value
ai_providerThe AI provider to use for content generation'anthropic'
api_keyThe API key for the selected AI providerNone
max_tokensThe maximum number of tokens to generate1024
temperatureThe temperature value to control the creativity of the generated content0.7

Common Use Cases​

  1. Generating Feature Descriptions: Use the AI-powered writing to create high-level descriptions of the tool's features and capabilities.
  2. Explaining Complex Concepts: Leverage the AI-generated content to provide detailed explanations of how the system works, including workflows and architectural details.
  3. Enhancing Usage Examples: Combine the AI-generated content with code examples to create comprehensive, user-friendly documentation.

Diagram Generation​

SourceScribe automatically generates various types of diagrams to visually illustrate the system's architecture, data models, and workflows. The supported diagram types include:

  • Sequence diagrams
  • Flowcharts
  • Class diagrams
  • State diagrams

How It Works​

Use mouse wheel to zoom • Click and drag to pan

Usage Examples​

from sourcescribe.engine.diagram import DiagramGenerator

generator = DiagramGenerator()
sequence_diagram = generator.generate_sequence_diagram(metadata)
print(sequence_diagram)

Configuration​

The diagram generation process can be configured using the following settings:

SettingDescriptionDefault Value
diagram_themesThe themes to use for the generated diagrams['default']
diagram_output_formatThe output format for the generated diagrams'png'
diagram_output_directoryThe directory to save the generated diagrams'docs/diagrams'

Common Use Cases​

  1. Illustrating Workflows: Use sequence diagrams to visually explain the key workflows and interactions within the system.
  2. Documenting Data Models: Leverage class diagrams to document the system's data models and their relationships.
  3. Depicting State Transitions: Employ state diagrams to illustrate the stateful behavior of the system's components.

Configuration Management​

SourceScribe provides a flexible configuration system to customize the documentation generation process. The configuration settings are defined in YAML format and can be loaded from a file or programmatically.

How It Works​

Use mouse wheel to zoom • Click and drag to pan

Usage Examples​

from sourcescribe.config.loader import ConfigLoader

config = ConfigLoader.load_config('path/to/config.yml')
print(config.ai_provider)

Configuration​

The configuration settings are defined in the following YAML structure:

ai_provider: 'anthropic'
api_key: 'your_api_key'
max_tokens: 1024
temperature: 0.7

include_patterns:
  - '*.py'
exclude_patterns:
  - '*_test.py'
max_file_size: 1048576

diagram_themes:
  - 'default'
diagram_output_format: 'png'
diagram_output_directory: 'docs/diagrams'

Common Use Cases​

  1. Customizing AI Provider: Switch between different AI providers (e.g., Anthropic, OpenAI) to generate content.
  2. Adjusting Diagram Settings: Modify the output format, themes, and directory for the generated diagrams.
  3. Filtering Source Files: Configure the include and exclude patterns to control which files are analyzed and documented.

Monitoring & Logging​

SourceScribe provides logging and monitoring capabilities to track the documentation generation process and handle any issues that may arise.

How It Works​

Use mouse wheel to zoom • Click and drag to pan

Usage Examples​

from sourcescribe.utils.logger import Logger

logger = Logger()
logger.info('Starting documentation generation')
try:
    # Generate documentation
except Exception as e:
    logger.error(f'Error generating documentation: {e}')

Configuration​

The logging and monitoring features can be configured using the following settings:

SettingDescriptionDefault Value
log_levelThe minimum log level to record'INFO'
log_fileThe path to the log file'sourcescribe.log'
log_formatThe format of the log messages'%(asctime)s [%(levelname)s] %(message)s'

Common Use Cases​

  1. Debugging Issues: Use the logging system to identify and troubleshoot any problems that occur during the documentation generation process.
  2. Monitoring Workflow: Track the progress of the documentation generation and receive notifications about its status.
  3. Integrating with CI/CD: Leverage the logging capabilities to integrate SourceScribe into your continuous integration and deployment pipelines.

Extensibility​

SourceScribe is designed to be extensible, allowing you to integrate new AI providers, add custom diagram types, or extend the core functionality.

Usage Examples​

from sourcescribe.engine.diagram import DiagramGenerator
from sourcescribe.engine.feature_generator import FeatureGenerator

class CustomAIProvider(AIProvider):

        pass

class CustomDiagramGenerator(DiagramGenerator):
    def generate_custom_diagram(self, metadata):
        # Implement custom diagram generation logic
        pass

class CustomFeatureGenerator(FeatureGenerator):
    def generate_custom_feature(self, metadata):
        # Implement custom feature generation logic
        pass

Configuration​

To use the custom extensions, update the configuration settings:

ai_provider: 'custom'
diagram_generators:
  - 'custom'
feature_generators:
  - 'custom'

Common Use Cases​

  1. Integrating New AI Providers: Add support for additional AI-powered writing assistants, such as GPT-3 or Hugging Face models.
  2. Extending Diagram Types: Implement custom diagram generators to visualize specific aspects of the system that are not covered by the built-in diagram types.
  3. Customizing Feature Generation: Develop custom feature generators to produce specialized documentation content tailored to your project's needs.

Conclusion​

SourceScribe is a powerful documentation generation tool that combines code analysis, AI-powered writing, and diagram generation to create comprehensive, user-centric technical documentation. With its flexible configuration system and extensibility, SourceScribe can be customized to fit the specific needs of your project and development workflow.

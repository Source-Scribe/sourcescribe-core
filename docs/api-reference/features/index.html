Features

SourceScribe Documentation

SourceScribe is a powerful documentation generation tool that automatically creates user-friendly technical documentation from source code. It supports a variety of programming languages and integrates with popular AI-based content generation services to enhance the documentation experience.

Features​

SourceScribe provides the following key features:

  1. Code Analysis & Documentation Generation
  2. AI-Powered Content Generation
  3. Configurable Documentation Styles
  4. Monitoring & Logging
  5. Extensible API Integration

Code Analysis & Documentation Generation​

SourceScribe analyzes your source code and automatically generates comprehensive technical documentation. It extracts relevant information, such as code structures, comments, and docstrings, to create user-centric documentation.

How It Works

The code analysis and documentation generation process in SourceScribe can be summarized as follows:

Use mouse wheel to zoom • Click and drag to pan
  1. Code Parser: SourceScribe's parser module analyzes the source code, extracting relevant information such as class definitions, function signatures, and docstrings.
  2. Feature Extractor: The feature extractor module processes the parsed data, identifying key features and capabilities within the codebase.
  3. Documentation Generator: Based on the extracted features, the documentation generator creates comprehensive, user-centric documentation, including detailed explanations, code examples, and visual diagrams.
  4. Rendered Documentation: The generated documentation is then rendered in a format suitable for consumption, such as Markdown or HTML.

Usage Examples

To generate documentation for your project, you can use the SourceScribe CLI:

sourcescribe generate --input-dir /path/to/your/project --output-dir /path/to/documentation

This command will analyze the source code in the specified input directory and generate the documentation in the output directory.

Configuration

SourceScribe's code analysis and documentation generation features can be configured using the following settings:

SettingDescriptionDefault Value
input_dirThe directory containing the source code to be analyzed.None
output_dirThe directory where the generated documentation will be saved.None
exclude_patternsA list of file patterns to exclude from the analysis.[]
languageThe programming language of the source code.python

You can set these configuration options in a YAML file or programmatically using the SourceScribe API.

Common Use Cases

  • Automatically generate documentation for a new software project or codebase.
  • Update documentation when the source code changes, ensuring the documentation remains up-to-date.
  • Provide comprehensive technical documentation for open-source or internal projects.

AI-Powered Content Generation​

SourceScribe integrates with AI-based content generation services, such as Anthropic and OpenAI, to enhance the documentation experience. These services can be used to generate high-quality, contextual content to complement the automatically generated documentation.

How It Works

SourceScribe's AI integration follows this process:

Use mouse wheel to zoom • Click and drag to pan
  1. The user requests AI-generated content within the SourceScribe documentation.
  2. SourceScribe sends the request to the configured AI provider, such as Anthropic or OpenAI.
  3. The AI provider processes the request and generates the relevant content.
  4. SourceScribe receives the generated content and includes it in the final documentation.

Usage Examples

To use the AI-powered content generation feature, you can configure the AI provider settings in SourceScribe:

from sourcescribe.config.models import AIProviderConfig

config = AIProviderConfig(
    provider="anthropic",
    api_key="your_anthropic_api_key",
    model="davinci",
    temperature=0.7,
    max_tokens=500
)

sourcescribe = SourceScribe(ai_provider_config=config)
sourcescribe.generate_documentation()

In this example, SourceScribe is configured to use the Anthropic AI provider with the specified settings.

Configuration

SourceScribe's AI-powered content generation feature can be configured using the following settings:

SettingDescriptionDefault Value
providerThe AI provider to use (e.g., "anthropic", "openai").None
api_keyThe API key for the selected AI provider.None
modelThe AI model to use for content generation.None
temperatureThe temperature parameter for the AI model.0.7
max_tokensThe maximum number of tokens to generate.500

You can set these configuration options in a YAML file or programmatically using the SourceScribe API.

Common Use Cases

  • Generate detailed explanations and examples for complex technical concepts.
  • Automatically create high-quality summaries and overviews for the documented features.
  • Enhance the documentation with AI-generated content to provide a more engaging and informative experience for users.

Configurable Documentation Styles​

SourceScribe allows you to customize the style and format of the generated documentation to match your project's branding and requirements.

How It Works

SourceScribe's documentation style configuration is managed through a set of settings that control the overall look and feel of the generated documentation.

Use mouse wheel to zoom • Click and drag to pan
  1. Theme: The overall visual theme of the documentation, such as "dark" or "light".
  2. Font: The font used throughout the documentation.
  3. Code Theme: The syntax highlighting theme for code examples.
  4. Logo: The project's logo, which will be displayed in the documentation.
  5. Header Image: An optional image to be displayed in the header of the documentation.
  6. Footer Text: Additional text to be displayed in the footer of the documentation.

Usage Examples

To configure the documentation style, you can use the DocumentationStyle class:

from sourcescribe.config.models import DocumentationStyle

style = DocumentationStyle(
    theme="dark",
    font="Roboto",
    code_theme="monokai",
    logo="path/to/logo.png",
    header_image="path/to/header.jpg",
    footer_text="© 2023 SourceScribe"
)

sourcescribe = SourceScribe(documentation_style=style)
sourcescribe.generate_documentation()

Configuration

The DocumentationStyle class provides the following configuration options:

SettingDescriptionDefault Value
themeThe visual theme of the documentation."light"
fontThe font used throughout the documentation."Arial"
code_themeThe syntax highlighting theme for code examples."github"
logoThe path to the project's logo image.None
header_imageThe path to an optional header image.None
footer_textAdditional text to be displayed in the footer.None

You can set these configuration options in a YAML file or programmatically using the SourceScribe API.

Common Use Cases

  • Align the documentation's visual style with the project's branding and design guidelines.
  • Provide a consistent and visually appealing documentation experience for users.
  • Customize the documentation to match the overall look and feel of the project.

Monitoring & Logging​

SourceScribe provides comprehensive monitoring and logging capabilities to help you track the documentation generation process and troubleshoot any issues that may arise.

How It Works

SourceScribe's monitoring and logging features are powered by the following components:

Use mouse wheel to zoom • Click and drag to pan
  1. When the user initiates the documentation generation process, SourceScribe logs the start of the process.
  2. During the generation process, SourceScribe logs progress updates and any relevant information.
  3. Upon completion of the documentation generation, SourceScribe logs the final status.
  4. The user can access the log output to monitor the process and troubleshoot any issues that may have occurred.

Usage Examples

To access the log output, you can use the SourceScribe CLI:

sourcescribe generate --input-dir /path/to/your/project --output-dir /path/to/documentation --log-level DEBUG

This command will generate the documentation and display the log output with a DEBUG log level.

Alternatively, you can access the log output programmatically:

from sourcescribe.utils.logger import get_logger

logger = get_logger()
logger.info("Starting documentation generation")
# SourceScribe generation code
logger.info("Documentation generation complete")

Configuration

SourceScribe's monitoring and logging features can be configured using the following settings:

SettingDescriptionDefault Value
log_levelThe log level for the documentation generation process."INFO"
log_fileThe path to the log file, if you want to save the logs to a file.None

You can set these configuration options in a YAML file or programmatically using the SourceScribe API.

Common Use Cases

  • Monitor the progress of the documentation generation process.
  • Troubleshoot issues that may occur during the generation process.
  • Analyze the log output to identify performance bottlenecks or other optimization opportunities.

Extensible API Integration​

SourceScribe provides an extensible API that allows you to integrate it with other tools and services, enabling a more seamless documentation workflow.

How It Works

SourceScribe's API integration follows a provider-based architecture, allowing you to easily add support for new providers or services.

Use mouse wheel to zoom • Click and drag to pan
  1. The APIProvider class defines the common interface for all API providers.
  2. SourceScribe includes built-in providers, such as AnthropicProvider and OpenAIProvider, which implement the API provider interface.
  3. You can create custom providers by extending the APIProvider class and implementing the required methods, allowing you to integrate SourceScribe with your own services or tools.

Usage Examples

To use the built-in API providers, you can configure them in SourceScribe:

from sourcescribe.api.factory import get_api_provider

provider = get_api_provider("anthropic", api_key="your_anthropic_api_key")
provider.generate_documentation(input_dir="/path/to/your/project", output_dir="/path/to/documentation")

To create a custom API provider, you can extend the APIProvider class:

from sourcescribe.api.base import APIProvider

class CustomProvider(APIProvider):
    def generate_documentation(self, input_dir, output_dir):
        # Implement custom documentation generation logic
        pass

    def generate_content(self, prompt, **kwargs):
        # Implement custom content generation logic
        pass

custom_provider = CustomProvider(api_key="your_custom_api_key")
custom_provider.generate_documentation(input_dir="/path/to/your/project", output_dir="/path/to/documentation")

Configuration

SourceScribe's API integration can be configured using the following settings:

SettingDescriptionDefault Value
providerThe API provider to use (e.g., "anthropic", "openai", "custom").None
api_keyThe API key for the selected provider.None

You can set these configuration options in a YAML file or programmatically using the SourceScribe API.

Common Use Cases

  • Integrate SourceScribe with your organization's custom content generation or documentation tools.
  • Extend SourceScribe's capabilities by adding support for new AI providers or other external services.
  • Streamline your documentation workflow by connecting SourceScribe to other tools in your ecosystem.

Conclusion​

SourceScribe is a powerful documentation generation tool that simplifies the process of creating comprehensive, user-centric technical documentation. With its feature-rich capabilities, including code analysis, AI-powered content generation, configurable documentation styles, monitoring and logging, and extensible API integration, SourceScribe helps you deliver high-quality documentation that enhances the user experience and streamlines your documentation workflow.

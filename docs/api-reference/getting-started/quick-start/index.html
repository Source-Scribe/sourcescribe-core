Quick Start

SourceScribe Quick Start Guide

First Steps​

To get started with SourceScribe, you'll need to install the package and set up your configuration. Here's how:

  1. Install SourceScribe:

    pip install sourcescribe

  2. Create a configuration file:

    • SourceScribe uses a YAML configuration file to specify your preferences and settings.
    • You can create a new configuration file by running: 
      sourcescribe init
    • This will create a sourcescribe.yml file in your current directory.
    • Open the file and customize the settings to your liking.

  3. Verify the installation:

    • Run the following command to ensure SourceScribe is installed and configured correctly: 
      sourcescribe run --dry-run
    • This will perform a dry run of the SourceScribe process without generating any output.

That's it! You're now ready to start using SourceScribe.

Hello World Example​

Let's walk through a simple example to see SourceScribe in action.

  1. Prepare your code:

    • Create a new Python file, e.g., example.py, with the following content: 
      def greet(name):
          """Greet the user by name."""
          return f"Hello, {name}!"

      if __name__ == "__main__":
          print(greet("World"))

  2. Run SourceScribe:

    sourcescribe run example.py

  3. Explore the output:

    • SourceScribe will generate a set of files in the docs/ directory, including:
      • index.md: The main documentation page
      • api.md: Documentation for the greet function
    • Open the index.md file in your preferred Markdown viewer or editor.

What Just Happened?​

When you ran the sourcescribe run example.py command, SourceScribe performed the following steps:

Use mouse wheel to zoom • Click and drag to pan
  1. SourceScribe parsed the example.py file and analyzed the Python code.
  2. Based on the code analysis, SourceScribe generated the necessary documentation, including:
    • An overview of the project
    • Documentation for the greet function
  3. SourceScribe then created Markdown files in the docs/ directory, which you can view and share with others.

Next Steps​

Now that you've seen a simple example, here are some next steps you can take:

  1. Explore the Configuration: Dive into the sourcescribe.yml file and customize the settings to fit your project's needs. You can adjust things like the output directory, file exclusions, and more.

  2. Document Your Entire Project: Run SourceScribe on your entire codebase to generate comprehensive documentation for your project.

  3. Integrate SourceScribe into Your Workflow: Consider adding SourceScribe to your continuous integration (CI) pipeline to automatically generate and update your project's documentation.

  4. Learn More: Check out the SourceScribe documentation for more advanced features and usage examples.

Happy documenting!

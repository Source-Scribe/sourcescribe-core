Configuration

SourceScribe Configuration

SourceScribe is a powerful documentation generation tool that allows you to create comprehensive, user-centric technical documentation from your source code. This guide covers the various configuration options available to customize the behavior and output of SourceScribe.

Configuration Files​

SourceScribe supports configuration via YAML files. By default, it looks for a sourcescribe.yml file in the current working directory. You can also specify a custom configuration file path using the --config command-line option.

The configuration file follows a simple structure with top-level keys for the various settings. Here's an example:

# sourcescribe.yml
output_dir: docs
include_patterns:
  - "**/*.py"
exclude_patterns:
  - "**/tests/**"
  - "**/venv/**"
providers:
  openai:
    api_key: YOUR_OPENAI_API_KEY
  anthropic:
    api_key: YOUR_ANTHROPIC_API_KEY
  ollama:
    api_key: YOUR_OLLAMA_API_KEY

Configuration Options​

SourceScribe supports the following configuration options:

OptionDescriptionDefault
output_dirThe directory where the generated documentation will be saved."docs"
include_patternsA list of glob patterns to include when scanning for source files.["**/*.py"]
exclude_patternsA list of glob patterns to exclude when scanning for source files.["**/tests/**", "**/venv/**"]
providersConfiguration for the various AI providers used by SourceScribe.See below

AI Provider Configuration​

SourceScribe supports multiple AI providers, each with their own set of configuration options. The available providers are:

  • openai: OpenAI's language models
  • anthropic: Anthropic's language models
  • ollama: Ollama's language models

For each provider, you'll need to specify the api_key that SourceScribe will use to authenticate with the provider's API.

Example provider configuration:

providers:
  openai:
    api_key: YOUR_OPENAI_API_KEY
  anthropic:
    api_key: YOUR_ANTHROPIC_API_KEY
  ollama:
    api_key: YOUR_OLLAMA_API_KEY

Environment Variables​

SourceScribe supports the following environment variables:

VariableDescriptionRequired
SOURCESCRIBE_CONFIGThe path to the SourceScribe configuration file.No
OPENAI_API_KEYThe API key for the OpenAI provider.Yes, if using OpenAI
ANTHROPIC_API_KEYThe API key for the Anthropic provider.Yes, if using Anthropic
OLLAMA_API_KEYThe API key for the Ollama provider.Yes, if using Ollama

Examples​

Basic Configuration​

output_dir: docs
include_patterns:
  - "**/*.py"
exclude_patterns:
  - "**/tests/**"
  - "**/venv/**"
providers:
  openai:
    api_key: YOUR_OPENAI_API_KEY

This configuration will:

  • Generate the documentation in the docs directory
  • Include all Python files in the project
  • Exclude the tests and venv directories
  • Use the OpenAI provider for language model generation

Customized Configuration​

output_dir: documentation
include_patterns:
  - "**/*.py"
  - "**/*.js"
exclude_patterns:
  - "**/node_modules/**"
  - "**/venv/**"
providers:
  openai:
    api_key: YOUR_OPENAI_API_KEY
  anthropic:
    api_key: YOUR_ANTHROPIC_API_KEY
  ollama:
    api_key: YOUR_OLLAMA_API_KEY

This configuration will:

  • Generate the documentation in the documentation directory
  • Include both Python and JavaScript files
  • Exclude the node_modules and venv directories
  • Use all three available providers (OpenAI, Anthropic, and Ollama)

Best Practices​

  • Use a version-controlled configuration file: Keep your SourceScribe configuration in your project's version control system to ensure consistency across different environments.
  • Leverage environment variables: Store sensitive information, such as API keys, in environment variables to keep them out of your configuration file.
  • Exclude test and temporary directories: Avoid generating documentation for test files and temporary directories to keep the output focused and relevant.
  • Start with a simple configuration: Begin with a basic configuration and gradually add more customizations as needed.
  • Document your configuration: Provide clear comments in your configuration file to explain the purpose of each setting.

By following these best practices, you can ensure that your SourceScribe configuration is maintainable, secure, and tailored to your project's specific needs.

Configuration

SourceScribe Configuration

SourceScribe is a powerful documentation tool that generates user-centric technical documentation from your source code. This guide covers the various configuration options available to customize the behavior and output of SourceScribe.

Configuration Files​

SourceScribe supports configuration via YAML files. By default, it looks for a sourcescribe.yml file in the current working directory. You can also specify a custom configuration file using the --config or -c command-line option.

The configuration file follows a simple structure with top-level keys for the various settings. Here's an example:

# sourcescribe.yml
output_dir: docs
include_patterns:
  - "**/*.py"
exclude_patterns:
  - "**/tests/**"
  - "**/venv/**"
providers:
  openai:
    api_key: your_openai_api_key
  anthropic:
    api_key: your_anthropic_api_key
  ollama:
    api_key: your_ollama_api_key

Configuration Options​

The following table outlines the available configuration options for SourceScribe:

OptionDescriptionDefault
output_dirThe directory where the generated documentation will be saved."docs"
include_patternsA list of file patterns (using glob syntax) to include in the documentation generation.["**/*.py"]
exclude_patternsA list of file patterns (using glob syntax) to exclude from the documentation generation.["**/tests/**", "**/venv/**"]
providersConfiguration for the various AI providers used by SourceScribe.See below

AI Provider Configuration​

SourceScribe supports multiple AI providers for code analysis and documentation generation. Each provider has its own set of configuration options:

OpenAI​

OptionDescriptionDefault
api_keyYour OpenAI API key.None
modelThe OpenAI model to use for code analysis."text-davinci-003"
temperatureThe temperature parameter for the OpenAI model.0.7
max_tokensThe maximum number of tokens to generate per response.2048

Anthropic​

OptionDescriptionDefault
api_keyYour Anthropic API key.None
modelThe Anthropic model to use for code analysis."claude-v1"
temperatureThe temperature parameter for the Anthropic model.0.7
max_tokensThe maximum number of tokens to generate per response.2048

Ollama​

OptionDescriptionDefault
api_keyYour Ollama API key.None
modelThe Ollama model to use for code analysis."ollama-v1"
temperatureThe temperature parameter for the Ollama model.0.7
max_tokensThe maximum number of tokens to generate per response.2048

Environment Variables​

SourceScribe supports the following environment variables for configuration:

VariableDescription
SOURCESCRIBE_CONFIG_FILEThe path to a custom configuration file.
OPENAI_API_KEYYour OpenAI API key.
ANTHROPIC_API_KEYYour Anthropic API key.
OLLAMA_API_KEYYour Ollama API key.

Examples​

Basic Configuration​

Here's a basic configuration that generates documentation for all Python files, excluding tests and virtual environments:

output_dir: docs
include_patterns:
  - "**/*.py"
exclude_patterns:
  - "**/tests/**"
  - "**/venv/**"

Configuring AI Providers​

This configuration uses all three AI providers (OpenAI, Anthropic, and Ollama) with custom settings:

output_dir: docs
include_patterns:
  - "**/*.py"
exclude_patterns:
  - "**/tests/**"
  - "**/venv/**"
providers:
  openai:
    api_key: your_openai_api_key
    model: text-davinci-003
    temperature: 0.5
    max_tokens: 1024
  anthropic:
    api_key: your_anthropic_api_key
    model: claude-v1
    temperature: 0.6
    max_tokens: 1024
  ollama:
    api_key: your_ollama_api_key
    model: ollama-v1
    temperature: 0.6
    max_tokens: 1024

Best Practices​

  • Use environment variables for sensitive information like API keys to avoid committing them to version control.
  • Experiment with different AI provider configurations to find the best balance of quality and performance for your project.
  • Regularly review and update the configuration as your project evolves to ensure the generated documentation remains relevant and accurate.
  • Consider using a centralized configuration management system (e.g., a shared YAML file) if multiple team members need to access the SourceScribe configuration.
